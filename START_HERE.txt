🎯 МОДЕЛЬ COURSE: ПОЛНАЯ СПЕЦИФИКАЦИЯ


  НАЗВАНИЕ:        courses
  ТИП:             Collection Type
  DRAFT & PUBLISH: ✅ Включено


  ПОЛЯ:

    • title          [String, обязательно]       Название курса (3-255 символов)
    • slug           [UID, обязательно]          URL ID (автогенерируется)
    • description    [RichText, обязательно]     Описание с HTML
    • category       [Enum, обязательно]         
      └─ programming, design, business, marketing, management, soft-skills, other
    • level          [Enum, обязательно]         
      └─ beginner, intermediate, advanced
    • duration       [Integer, обязательно]      Минуты
    • instructor     [String, обязательно]       1-255 символов

  АВТОМАТИЧЕСКИЕ ПОЛЯ:
    • id             [Integer]                   Уникальный ID
    • createdAt      [Timestamp]                 Создано
    • updatedAt      [Timestamp]                 Обновлено
    • publishedAt    [Timestamp]                 Опубликовано

🔌 API ENDPOINTS (ПОСЛЕ УСТАНОВКИ)


  GET    /api/courses             Получить все курсы
  GET    /api/courses/:id         Получить курс по ID
  POST   /api/courses             Создать курс
  PUT    /api/courses/:id         Обновить курс
  DELETE /api/courses/:id         Удалить курс
  POST   /api/courses/:id/publish Опубликовать


  📝 ПРИМЕР:

  curl -X POST http://localhost:1337/api/courses \
    -H "Content-Type: application/json" \
    -d '{
      "data": {
        "title": "JavaScript Basics",
        "description": "<p>Learn JS</p>",
        "category": "programming",
        "level": "beginner",
        "duration": 600,
        "instructor": "John Doe"
      }
    }'

⏳ СЛЕДУЮЩИЕ ШАГИ (РЕКОМЕНДУЕМЫЕ)


  После успешной установки Course модели:

  1. Создать модели:
     • Lesson (уроки в курсе)
     • Module (модули курса)
     • Enrollment (регистрация студентов)
     • User (студенты/инструкторы)

  2. Установить связи между моделями:
     • Course → hasMany Lessons
     • Course → hasMany Enrollments
     • Enrollment → belongsTo User

  3. Настроить Permissions & Roles:
     • Студент может просматривать
     • Инструктор может редактировать
     • Админ может управлять

  4. Добавить функциональность:
     • File uploads для видео/документов
     • Система оценок
     • Уведомления студентам
